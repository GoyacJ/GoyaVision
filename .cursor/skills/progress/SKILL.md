# 开发进度更新指南

用于同步 `docs/development-progress.md` 的状态与说明。

## 何时使用

✅ **推荐场景**：
- 功能完成或进入新阶段
- 发现阻塞或技术债务
- 需要更新里程碑进度

❌ **不适用场景**：
- 只需要查看开发进度（使用 Read 工具）
- 小的 Bug 修复（通常不需要更新进度）

## 状态标记

- ✅ **已完成** - 功能已实现、测试通过、文档更新
- 🚧 **进行中** - 正在开发，部分功能可用
- ⏸️ **待开始** - 计划中但尚未开始
- ⚠️ **阻塞** - 遇到阻塞问题，需要解决
- 🔄 **重构中** - 功能可用但正在重构优化

## 更新步骤

1. **找到对应模块与功能条目**
   - 在 `docs/development-progress.md` 中找到相关功能
   - 确认当前状态和说明

2. **更新状态与说明列**
   - 将状态从一个阶段更新到另一个阶段
   - 更新说明列，描述已完成的功能或当前进度

3. **如有阻塞/风险，补充说明**
   - 在说明列中注明阻塞原因
   - 记录技术债务或已知问题

## 更新示例

### 功能完成示例

**变更前**:
```markdown
| 媒体资产管理 | 🚧 | CRUD 已完成，搜索功能开发中 |
```

**变更后**:
```markdown
| 媒体资产管理 | ✅ | 已完成 CRUD、标签系统、派生追踪、搜索过滤功能 |
```

### 开始新功能示例

**变更前**:
```markdown
| 算子管理 | ⏸️ | 待开始 |
```

**变更后**:
```markdown
| 算子管理 | 🚧 | CRUD 已完成，分类管理开发中 |
```

### 功能阻塞示例

**变更前**:
```markdown
| 复杂工作流 | 🚧 | DAG 并行执行引擎开发中 |
```

**变更后**:
```markdown
| 复杂工作流 | ⚠️ | 阻塞：DAG 并行执行引擎设计评审中，等待架构决策 |
```

## 更新时机

### 必须更新的情况

- ✅ **完成主要功能** - 功能从 🚧 进行中 → ✅ 已完成
- 🚧 **开始新功能** - 功能从 ⏸️ 待开始 → 🚧 进行中
- ⚠️ **遇到阻塞** - 功能被阻塞，需要标记并说明原因

### 可选更新的情况

- 完成子功能时更新"说明"列
- 解决技术债务时更新备注
- 调整优先级或计划变更时

## 配合其他工具

开发进度更新通常与以下工具一起使用：

1. `/dev-done` - 完成开发后自动提醒更新进度
2. `/commit` - 提交时在 commit message 中引用进度变更
3. `CHANGELOG.md` - 进度更新后同步到变更日志

## 注意事项

- 状态标记要准确反映真实情况
- 说明要简洁但足够详细
- 重要功能完成时添加完成时间
- 遇到阻塞立即标记并说明原因
- 定期回顾和更新整体进度
